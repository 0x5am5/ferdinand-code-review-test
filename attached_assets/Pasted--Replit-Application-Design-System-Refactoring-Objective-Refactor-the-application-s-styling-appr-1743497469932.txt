# Replit Application Design System Refactoring

## Objective
Refactor the application's styling approach from inline utility classes to a structured SCSS system using semantic variables that are built from core design tokens. Implement component-specific styling files that use these variables consistently.

## Tasks

### 1. Design Token System Implementation
- Create a structured SCSS design token system with two levels:
  - **Core Variables**: Raw color values (brand colors, neutrals, interactive states)
  - **Semantic Variables**: Purpose-specific tokens mapped to core variables
- Make core variables editable within the design builder interface
- Ensure semantic variables properly reference and extend the core variables

### 2. File Structure Organization
- Create the following directory structure for styling:
```
src/
  styles/
    tokens/
      _core.scss        # Raw/core variables (colors, spacing, etc.)
      _semantic.scss    # Semantic variables mapped to core variables
    mixins/
      _responsive.scss  # Media query mixins
      _typography.scss  # Typography mixins
    _reset.scss         # CSS reset/normalize
    global.scss         # Main import file
  components/
    ComponentName/
      ComponentName.tsx
      ComponentName.module.scss   # Component-specific styles
```

### 3. Component Style Refactoring
- For each component:
  1. Create a dedicated ComponentName.module.scss file
  2. Import the semantic variables (`@import '../../styles/tokens/semantic';`)
  3. Define component-specific styles using semantic class names
  4. Replace inline utility classes with references to the SCSS module

### 4. From Utility Classes to Semantic CSS
- Replace utility classes like `bg-gray-100 text-gray-800 p-4` with semantic classes
- Create meaningful class names based on component purpose rather than appearance
- Example transformation:
  - Before: `<div className="bg-gray-100 p-4 rounded-lg shadow-sm">`
  - After: `<div className={styles.card}>` with `.card` defined in SCSS

### 5. Variable Implementation Details
- Assign appropriate values to all core color variables
- Ensure all semantic variables correctly map to core variables
- Add comments explaining the purpose of each variable group
- Implement the provided variable structure exactly as shown in the variable lists

### 6. Special Instructions
- Maintain all existing component functionality while refactoring styles
- Add comments explaining the semantic purpose of classes
- Do not remove any components or features while refactoring
- Use nested SCSS where appropriate for component states
- Ensure responsive behavior is preserved through mixins

## Example Component Refactoring

Card component - before:
```jsx
const Card = ({ children }) => (
  <div className="rounded-lg border border-gray-200 bg-white p-6 shadow-sm hover:border-gray-300">
    {children}
  </div>
);
```

Card component - after:
```jsx
import styles from './Card.module.scss';

const Card = ({ children }) => (
  <div className={styles.card}>
    {children}
  </div>
);
```

With Card.module.scss:
```scss
@import '../../styles/tokens/semantic';

.card {
  border-radius: 8px;
  border: 1px solid $color-border-default;
  background-color: $color-background-default;
  padding: 1.5rem;
  box-shadow: 0 1px 3px $color-elevation-shadow-1;
  
  &:hover {
    border-color: $color-border-strong;
  }
}
```
